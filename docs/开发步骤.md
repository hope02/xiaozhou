# 小舟浏览器（鸿蒙）开发步骤

> 目标：你可以按步骤实现并同步调试，每一步都能看到阶段性成果。

## 0. 项目初始化与基线
### 要做什么
- 建立鸿蒙工程基础结构（UI 层、浏览核心层、数据层、同步层）
- 配置基础依赖与统一日志方案
- 搭建全局路由与设置页入口

### 调试与验收
- 应用可正常启动，无崩溃
- 首页可显示占位 UI（搜索框、工具栏、标签入口）
- 日志系统可输出 `info / warn / error`

---

## 1. 实现基础浏览能力（MVP 第一优先）
### 要做什么
- 地址栏：输入 URL 或关键词
- 页面容器：加载网页、前进、后退、刷新、停止
- 标签页：新建、切换、关闭
- 页面状态：加载中、失败、空白页提示

### 调试与验收
- 输入 `https://www.baidu.com` 可打开页面
- 输入普通关键词可走默认搜索
- 快速切换 3-5 个标签页不出现明显卡顿
- 弱网时能看到明确错误提示并支持重试

---

## 2. 实现书签功能（MVP 第一优先）
### 要做什么
- 从当前页面一键添加书签
- 书签列表：查看、编辑、删除
- 书签文件夹：创建与移动（先做一级目录）
- 首页常用书签区展示

### 调试与验收
- 新增/编辑/删除后重启应用，数据仍存在
- 相同 URL 重复收藏有去重或合理提示
- 文件夹内外移动后，展示与存储一致

---

## 3. 实现历史记录（MVP 第二优先）
### 要做什么
- 页面访问写入历史（标题、URL、时间）
- 历史页支持搜索和删除
- 提供“一键清空历史”

### 调试与验收
- 打开多个站点后可在历史页看到记录
- 关键字可检索到目标历史
- 清空后列表为空，重启应用后不回流

---

## 4. 抽象本地数据模型（为同步做准备）
### 要做什么
- 统一书签数据结构（`id / title / url / folderId / updatedAt / deletedFlag`）
- 增加版本字段（`schemaVersion`）
- 统一数据访问接口（Repository）

### 调试与验收
- 本地新增/更新/删除都能正确维护 `updatedAt`
- 软删除与真实删除策略明确，并在 UI 表现一致

---

## 5. 接入 WebDAV（连接与鉴权）
### 要做什么
- 设置页新增 WebDAV 配置项：地址、用户名、密码、目录
- 增加“测试连接”按钮
- 保存前做参数校验与连通性检查

### 调试与验收
- 正确配置可测试通过并保存
- 错误用户名/密码时提示鉴权失败
- URL 非 HTTPS 时（按你的策略）阻止或告警

---

## 6. 书签同步 MVP（核心）
### 要做什么
- 云端文件设计：例如 `bookmarks.json`
- 首次同步：本地空时拉取云端；云端空时上传本地
- 日常同步：双向增量（按 `updatedAt`）
- 触发时机：手动同步 + 启动自动同步（可开关）

### 调试与验收
- 两台设备同一账号能互相看到新书签
- 同步结果页显示成功/失败条数
- 断网时不崩溃，恢复后可再次同步成功

---

## 7. 冲突处理与恢复
### 要做什么
- 冲突判定：同一 `id` 在本地和云端都被修改
- MVP 策略：最后修改时间优先（LWW）
- 记录冲突日志与同步日志
- 失败重试：指数退避

### 调试与验收
- 人工制造冲突时，结果符合 LWW 规则
- 同步失败后可手动重试恢复
- 日志能定位失败原因（网络、鉴权、格式错误）

---

## 8. UI 打磨（简洁大方）
### 要做什么
- 首页布局精简：搜索 + 常用书签 + 最近访问
- 配色与圆角统一，减少视觉噪音
- 跟随系统深色模式
- 动效轻量化（避免过重动画）

### 调试与验收
- 深浅色切换时 UI 无错位
- 常用操作路径不超过 2 次点击
- 页面在常见分辨率下布局稳定

---

## 9. 稳定性与性能调试
### 要做什么
- 启动性能优化：延迟初始化非核心模块
- 浏览与切页内存观察，避免泄漏
- 同步链路超时与重试参数调优

### 调试与验收
- 连续浏览 20+ 页面不出现明显内存异常
- 冷启动与首屏时间达到你的目标值
- 同步在弱网下成功率可接受

---

## 10. 发布前检查
### 要做什么
- 完成核心回归用例（浏览、书签、同步、历史）
- 清理无用日志与调试开关
- 完善隐私说明与权限说明

### 调试与验收
- 核心流程全通过
- 无阻塞级崩溃与明显功能缺陷
- 可进入内测或首发发布流程

---

## 建议调试节奏（你可以直接照着跑）
- 每完成 1 个大步骤就打一个可回退版本标签（本地）
- 每天至少做一次“全链路同步测试”（新增、编辑、删除、冲突）
- 每次改同步逻辑后，先跑单设备，再跑双设备联调
- 问题记录建议按：`现象 -> 复现步骤 -> 预期 -> 实际 -> 日志`

## 建议优先级
- P0：浏览基础 + 书签 + WebDAV 配置 + 手动同步
- P1：自动同步 + 历史记录 + 冲突日志
- P2：下载管理 + 阅读体验增强 + 个性化主页
